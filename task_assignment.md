# DrivrBidRent - Task Assignment Document

## Project Overview
DrivrBidRent is a full-stack web application for vehicle auctions and rentals, supporting role-based features for Buyers, Sellers, Auction Managers, Admins, and Mechanics. The platform enables secure vehicle listing, bidding, renting, and management, with dashboards and workflows tailored to each user type.

## Technical Responsibilities Breakdown
- **Frontend:** EJS templates for dynamic pages, CSS for styling, JavaScript for interactivity.
- **Backend:** Node.js and Express.js for server logic, routing, and API creation.
- **Database:** MongoDb for storing user, vehicle, auction, rental, and notification data.
- **Full-Stack Collaboration:** All members contributed across frontend and backend, ensuring integration of all modules.

---

## Data Flow Summary
1. **Vehicle Listing:** Sellers add vehicles for auction/rental, stored in the database.
2. **Approval & Assignment:** Auction managers approve listings and assign mechanics for inspection.
3. **Buyer Interaction:** Buyers view approved vehicles, place bids, and book rentals.
4. **Booking & Bidding:** System tracks bids, bookings, and sends notifications to relevant users.
5. **Admin Monitoring:** Admin oversees all activities, manages users, and reviews analytics.
6. **Mechanic Updates:** Mechanics update inspection status, which is reflected in auction workflows.
